## üîç Overview

The Get External Company API enables partners to retrieve detailed company information including comprehensive investor matching data and fit scores. This endpoint provides instant access to company profiles with real-time investor compatibility analysis based on multiple business metrics.

## üìç Endpoint Details

<Card title="GET /api/v1/company" icon="arrow-right">

  **Method:** `GET`

  **URL:** `https://prodserver.diligentsia.co.uk/api/v1/company?ukRegNo=03237591`

  **Authentication:** Required (X-API-Key header)

  **Query Parameters:**
  - `ukRegNo` (string, required) - UK company registration number (6-8 alphanumeric characters)
  - Example: `03237591`

  **Note:** Both `/api/v1/company?ukRegNo=03237591` and `/api/v1/company/?ukRegNo=03237591` (with trailing slash) are supported.

</Card>

## üîë Query Parameters

| Parameter | Type | Required | Description | Example |
|-----------|------|----------|-------------|---------|
| `ukRegNo` | string | **Yes** | UK company registration number (6-8 alphanumeric characters) | `03237591` |

<Info>
  The UK registration number should be provided as a query parameter without spaces or special characters. The system will automatically normalize the format. Both `/api/v1/company?ukRegNo=03237591` and `/api/v1/company/?ukRegNo=03237591` (with trailing slash) are supported.
</Info>

| Header | Value | Required | Description |
|--------|-------|----------|-------------|

## üì• Response Format

### ‚úÖ Success Response (200)

The response includes comprehensive company data with investor matching results:

<Tabs>
  <Tab title="Company Information">
    
    ### Core Company Data

    | Field | Type | Description |
    |-------|------|-------------|
    | `companyId` | string | Unique identifier for the company |
    | `userId` | string | Associated user identifier |
    | `companyName` | string | Company name from Companies House |
    | `ukRegNo` | string | UK registration number |
    | `partnerId` | string | Partner organization ID |
    | `partnerName` | string | Partner organization name |
    
  </Tab>
  
  <Tab title="InvestorMatch Status">
    
    ### Match Completion Status
    
    | Field | Type | Description |
    |-------|------|-------------|
    | `investorMatchCompleted` | boolean | Whether all 6 InvestorMatch questions are answered |
    
    <Note>
      When `investorMatchCompleted` is `true`, the response includes a comprehensive `fitReport` with investor matching data.
    </Note>
    
  </Tab>
  
  <Tab title="Fit Report">
    
    ### Investor Matching Results
    
    The `fitReport` object contains:
    
    | Field | Type | Description |
    |-------|------|-------------|
    | `investorId` | string | Best matched investor ID |
    | `investorName` | string | Name of the best matched investor firm |
    | `fitScore` | number | Raw fit score (0-6) |
    | `fitScorePercentage` | number | Percentage score (0-100) |
    | `priorityScore` | number | Priority ranking score |
    | `completenessScore` | number | Data completeness score |
    | `breakdown` | object | Detailed scoring by criteria |
    | `investorCriteria` | object | Investor's preferences |
    | `companyProfile` | object | Company's InvestorMatch profile data |
    | `fitReportPath` | string | URL to the generated fit report PDF |
    
    ### Scoring Details

    The breakdown object includes:

    | Field | Type | Description |
    |-------|------|-------------|
    | `sectorScore` | number | Sector alignment score (0-1) |
    | `stageScore` | number | Stage alignment score (0-1) |
    | `sweetSpotScore` | number | Investment amount alignment score (0-1) |
    | `annualRevenuesScore` | number | Revenue alignment score (0-1) |
    | `annualProfitsScore` | number | Profit alignment score (0-1) |
    | `regionScore` | number | Regional alignment score (0-1) |
    
  </Tab>

  <Tab title="KPI Report">

    ### Key Performance Indicators

    The response includes a `kpiReport` array with detailed KPIs and a `kpiReportPath` for the PDF report:

    | Field | Type | Description |
    |-------|------|-------------|
    | `kpiReport` | array | Array of KPI objects with metrics |
    | `kpiReportPath` | string | URL to the generated KPI report PDF |

    Each KPI object contains:

    | Field | Type | Description |
    |-------|------|-------------|
    | `title` | string | KPI title |
    | `name` | string | Detailed KPI description |
    | `calculation` | string | How the KPI is calculated |
    | `metric` | string | Expected ranges and benchmarks |

  </Tab>
</Tabs>

## üéØ Scoring Breakdown

### Understanding Fit Scores

The fit scoring system evaluates compatibility across 6 key dimensions:

<AccordionGroup>
  <Accordion title="Sector Score (0-1)">
    - **1.0:** Perfect match - Company sector aligns with investor preferences
    - **0.5:** Partial match - Some sector alignment
    - **0.0:** No match - Sectors don't align
  </Accordion>
  
  <Accordion title="Stage Score (0-1)">
    - **1.0:** Perfect match - Company stage matches investor focus
    - **0.5:** Partial match - Some stage alignment
    - **0.0:** No match - Stage doesn't align
  </Accordion>
  
  <Accordion title="Sweet Spot Score (0-1)">
    - **1.0:** Perfect match - Investment amount sought matches investor range
    - **0.5:** Partial match - Some overlap in investment ranges
    - **0.0:** No match - Investment ranges don't align
  </Accordion>
  
  <Accordion title="Annual Revenues Score (0-1)">
    - **1.0:** Perfect match - Company revenues match investor criteria
    - **0.5:** Partial match - Some revenue alignment
    - **0.0:** No match - Revenue ranges don't align
  </Accordion>
  
  <Accordion title="Annual Profits Score (0-1)">
    - **1.0:** Perfect match - Company profits match investor criteria
    - **0.5:** Partial match - Some profit alignment
    - **0.0:** No match - Profit ranges don't align
  </Accordion>
  
  <Accordion title="Region Score (0-1)">
    - **1.0:** Perfect match - Company region matches investor focus
    - **0.5:** Partial match - Some regional alignment
    - **0.0:** No match - Regions don't align
  </Accordion>
</AccordionGroup>

<CardGroup cols={2}>
  <Card title="Fit Score Percentage" icon="percentage">

  </Card>
</CardGroup>
