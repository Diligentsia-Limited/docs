## üöÄ Overview

The Company Registration API enables partners to seamlessly register UK companies and automatically match them with potential investors. This powerful endpoint integrates with Companies House data and provides instant investor compatibility scoring based on multiple business metrics, along with comprehensive KPI reports for business performance tracking.

## üìç Endpoint Details

<Card title="POST /api/v1/company/create-with-investormatch" icon="paper-plane">
  
  **Method:** `POST`
  
  **URL:** `https://prodserver.diligentsia.co.uk/api/v1/company/create-with-investormatch`
  
  **Authentication:** Required (X-API-Key header)
    
</Card>

## üéõÔ∏è Request Parameters (Optional)

These optional flags control the API behavior:

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `sendEmail` | boolean | No | `true` | Controls whether to send activation email to the user |
| `backgroundProcess` | boolean | No | `false` | Process company creation asynchronously in the background |

<Info>
**Email Behavior:**
- When `sendEmail` is `true` (default): User receives an activation email with login instructions
- When `sendEmail` is `false`: API returns a `loginLink` in the response for direct access
</Info>

<Warning>
**Background Processing:**
- When `backgroundProcess` is `true`: Company, user, and reports are created asynchronously. API returns immediately with a confirmation message
- When `backgroundProcess` is `false` (default): API processes synchronously and returns complete results including investor matches and reports
</Warning>

## üìù Request Body Structure

The request body consists of three main sections:

<Tabs>
  <Tab title="User Information">

    <Note>
      **Optional Section** - If not provided, a guest user will be created
    </Note>

    ### User Fields

    | Field | Type | Required | Description |
    |-------|------|----------|-------------|
    | `email` | string | No | User's email address for account creation |
    | `firstName` | string | No | User's first name |
    | `lastName` | string | No | User's last name |

  </Tab>
  
  <Tab title="Company Details">
    
    <Warning>
      **Required Section** - All company fields must be provided
    </Warning>
    
    ### Company Fields
    
    | Field | Type | Required | Description | Validation |
    |-------|------|----------|-------------|------------|
    | `ukRegNo` | string | **Yes** | UK company registration number | 6-8 alphanumeric characters |
    | `companyWebsite` | string | **Yes** | Company website URL | Valid URL format |
    | `company5Words` | string | **Yes** | Company description | Minimum 5 words |
    
    <Tip>
      UK registration numbers are automatically normalized (spaces removed, uppercase)
    </Tip>
    
  </Tab>
  
  <Tab title="InvestorMatch Data">

    <Note>
      **Smart Auto-fill** - Some fields are auto-filled if not provided or invalid
    </Note>

    ### InvestorMatch Fields

    | Field | Type | Required | Description |
    |-------|------|----------|-------------|
    | `sector` | enum | **Yes** | Business sector |
    | `stage` | enum | No | Company development stage - **Auto-calculated** from `annualRevenues` if not provided or invalid |
    | `annualRevenues` | enum | No | Current year revenue range - **Defaults to "Nil"** if not provided or empty |
    | `annualProfits` | enum | **Yes** | Current year profit range |
    | `region` | enum | No | UK operating region - **Defaults to "UK"** if not provided, empty, or invalid |
    | `sweetSpot` | enum | **Yes** | Investment amount sought |

    <Tip>
      **Auto-fill Logic:**
      - **`annualRevenues`**: Automatically set to "Nil" if missing or empty
      - **`stage`**: Auto-calculated based on revenue:
        - "Nil" or 0 revenue ‚Üí **Pre-Revenue**
        - Revenue > 0 and < ¬£5m ‚Üí **Post-Revenue Early Stage**
        - Revenue ‚â• ¬£5m ‚Üí **Mature**
      - **`region`**: Defaults to "UK" if missing, empty, or invalid. AI normalization is attempted for invalid region values
    </Tip>

  </Tab>
</Tabs>

## ‚úÖ Validation Rules

### üè¢ Company Validation

<AccordionGroup>
  <Accordion title="UK Registration Number">
    - **Format:** 6-8 alphanumeric characters
    - **Pattern:** `/^[0-9A-Z]{6,8}$/`
    - **Auto-transformation:** Spaces removed, converted to uppercase
    - **Examples:** `12345678`, `AB123456`, `SC123456`
  </Accordion>
  
  <Accordion title="Company Website">
    - **Format:** Valid URL
    - **Protocol:** Optional (http/https added if missing)
    - **Examples:** `example.com`, `https://example.co.uk`
  </Accordion>
  
  <Accordion title="Company Description">
    - **Minimum:** 5 words
    - **Separator:** Space-separated words
    - **Example:** `Software development consulting services company`
  </Accordion>
</AccordionGroup>
